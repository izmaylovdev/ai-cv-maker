# AI CV Maker

An AI-powered CV generation platform built as an Nx monorepo. Users authenticate, fill in a profile, and get a tailored CV generated by an LLM — downloadable as a PDF.

## Architecture

| App | Tech | Description |
|-----|------|-------------|
| `ui-angular` | Angular + Nginx | Frontend SPA (auth, profile, CV pages) |
| `cv-api` | ASP.NET Core 9 + PostgreSQL | REST API — auth, user profiles, CV management, PDF export |
| `llm-service` | Python / FastAPI + LangChain | AI microservice — generates CV content via a configurable LLM |
| `infra` | Terraform | Cloud infrastructure |

```
Browser → ui-angular (Nginx) → cv-api (.NET) → PostgreSQL
                                     ↓
                              llm-service (Python/FastAPI)
```

## Getting started

### Prerequisites

- Docker & Docker Compose
- Node.js (for local frontend dev)

### Run with Docker Compose

Copy the example env file and fill in your values:

```sh
cp .env.example .env
```

Start all services:

```sh
docker compose up --build
```

| Service | URL |
|---------|-----|
| UI | http://localhost:4200 |
| CV API | http://localhost:5050 |
| LLM service | http://localhost:8080 |
| PostgreSQL | localhost:5433 |

### Local frontend dev

```sh
npx nx serve ui-angular
```

## LLM providers

Set `LLM_PROVIDER` in `.env` to switch backends:

| Provider | `LLM_PROVIDER` | Required env vars |
|----------|---------------|-------------------|
| Google Gemini (default) | `google` | `GOOGLE_API_KEY` |
| OpenAI / LM Studio | `openai` | `OPENAI_BASE_URL`, `OPENAI_API_KEY` |
| Microsoft Foundry (Claude) | `foundry` | `FOUNDRY_API_KEY`, `FOUNDRY_BASE_URL` or `FOUNDRY_DEPLOYMENT_NAME` |

> **Tip:** Point `OPENAI_BASE_URL=http://host.docker.internal:1234/v1` to run against a local LM Studio instance.

## Environment variables

See `.env.example` for the full list. Key variables:

```
POSTGRES_USER / POSTGRES_PASSWORD / POSTGRES_DB
JWT_SECRET
GOOGLE_CLIENT_ID          # Google OAuth
LLM_PROVIDER              # google | openai | foundry
LLM_MODEL                 # override default model name
LLM_TEMPERATURE           # default 0.4
```
